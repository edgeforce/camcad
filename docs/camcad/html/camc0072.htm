Database ClassNet: Connections are allowed to be made in any order and “T” connections are also allowed. Although rules must be followed for trace routing and via placement they are usually very loose, thus allowing the designer much more freedom in the routing process.

Board Outline ClassReal: Board outlines in this CAD format are true board outline entities.  CAMCAD supports true board outlines.

Component ClassLibrary: Library driven CAD systems have a component library section within the CAD data which defines all components.

Importing Veribest into CAMCAD

File(s) Used For Import

Files Generated by VB2RSI from the VeriBest system:

•eifo_cor.txt — EIF Correspondence file

•eifo_tec.txt — Job Parameters Technology file

•eifo_pad.txt — Pad Table file

•eifo_pin.txt — Pin Table file

•eifo_cel.txt — Cell Library file

•eifo_des.txt  — VeriBest PCB Design File(design.vbd)

•eifo_clr.txt — Job Parameters Clearances file

•netlist.kyn — Keyin Netlist (renamed from Netlist.txt or Netlist.aug)

How To Produce File(s)

For all CAMCAD products purchased with the VeriBest EIF import option, RSI will provide a zip archive with the utilities necessary to produce the EIF design files from the VeriBest design.

The zip archive VB972RSI.ZIP, contains a VB2RSI.doc file which documents the usage of the VB2RSI.exe and NETLIST.exe files. Follow the instructions to produce the EIF files for use with CAMCAD.

Notes On File Generation

Only one file, the EIFO_COR.txt file (same as the VeriBest Correspondence file), needs to be imported into CAMCAD. This file calls all of the other files in their proper order so that they are all imported correctly into CAMCAD. These should all reside in a design directory. The NETLIST.txt or NETLIST.aug file must be renamed to NETLIST.kyn and reside in the design directory.

The file import process is incremental; if you need to import any extra files after the correspondence file is imported (such as a netlist file) you can select File > Import > Veribest (EIF) Read again, and import the extra file.

Items controlled by the Eif.IN file

.LAYERATTR

This command sets the default layers which will be identified as PAD TOP, PAD BOTTOM, and PAD INNER.

.MIRRORLAYER

Establishes default layer mirroring.

.UNNAMEDNET

When nets are found which begin with the string given, CAMCAD will rename that net to $$__device.pin, where device.pin is the first device pin on that net.
